# game-factory UI/UX 設計（北極星ワイヤーフレーム）

Status: 確定（2026-07-20 レビュー済み）
位置づけ: Stage 4 完了時点の UI の最終形（北極星）を先に固定し、Stage 1 の最小 UI を「この部分集合」として作るための設計。**振る舞いの正本は openspec/**（承認前に確定しない等の契約はそちら）。この文書はレイアウト・遷移・表示の正本。

## 0. 決定事項（2026-07-20 ユーザー確認済み）

- **run 詳細は 2 ペイン統合**: 左タイムライン＋右プレビュー/差分ペイン。タブ分割・チャット風 1 カラムは不採用。
- **承認はインラインカード＋上部バナー**: 承認要求はタイムライン内のカードとして歴史に残し、承認待ちの間は画面上部にバナーを固定。モーダルは使わない。
- **進捗はフルストリーミング**: Agent のテキスト出力・Tool 呼び出し（引数要約・結果）をリアルタイムでタイムラインに流す。
- **PC ファースト**: 個人用ツールのためレスポンシブは後回し（1280px 以上を想定）。
- デザイントークンは Orange テンプレート（`~/.claude/skills/dashboard-design/tokens.md`）に従う。

### タイムラインの表示規則（RunEvent 種別 → 表示）

| RunEvent 種別 | 表示 | 色 |
|---|---|---|
| plan | 計画カード（生成ファイル一覧・仕様要約・想定 step・承認/却下ボタン） | 枠 `orange.600`、承認済みは `green.600` チェック |
| approval | 「計画承認（あなた）」「差分承認（あなた）」の監査行 | `green.600` |
| tool_call | Tool 名＋引数要約＋結果要約（クリックで全文展開） | 通常 `gray.700` |
| tool_denied | 拒否理由付きの警告行 | `red.600` アイコン |
| agent_text | Agent の発話（ストリーミングで文字が流れる） | `text.label` の吹き出し |
| halt | 停止カード（原因・その時点の step/費用・[再開] ボタン(Stage 3〜)） | `red.600` 枠 |
| cost | タイムラインには出さない（ヘッダーと費用タブに集約） | — |

- 承認カード・停止カードだけがカード（枠付き）で、他は軽い行。**重要なものだけ視覚的に重くする**（ガイドブックの強弱原則）。
- ストリーミングは Stage 1 では RunEvent の 1–2 秒 polling で実装し（見た目はストリーミング）、Stage 3 の Job 化と同時に SSE へ置き換える。UI コンポーネントはどちらでも動くよう「イベント配列の末尾追加」だけを契約にする。

## 4. 承認フローの遷移（UI 視点）

```
[計画カード]──✔ 承認──▶ GENERATING（タイムラインに承認行が残る）
     │
     └──✖ 却下──▶ 却下理由の入力（カード内、任意）──▶ REJECTED

[差分タブ (Stage 4)]──✔ 承認──▶ git commit（確定）──▶ DONE
     │
     ├──✖ 却下──▶ workspace 巻き戻し──▶ REJECTED
     └──↩ コメント付き再実行──▶ コメントが Agent への追加入力になり GENERATING へ差し戻し
```

- 承認・却下・再実行コメントはすべて RunEvent（approval）として記録され、タイムラインに残る（監査ログ＝UI の単一情報源）。
- 「ユーザー承認前に成果物を確定しない」（openspec approval-flow）を UI で表現する: PREVIEW_READY の間、右ペイン上部に `未確定` の淡色バッジを常時表示し、差分承認で初めて消える。

## 5. M1: 新規 Run 作成モーダル

```
┌───────────── 新規 Run ─────────────┐
│ 企画（自由テキスト）                  │
│ ┌────────────────────────────────┐ │
│ │ 猫が落ちてくる魚をキャッチする      │ │
│ │ 反射神経ゲーム。レトロな…          │ │
│ └────────────────────────────────┘ │
│ 上限: $1.00 / 30 step（既定値・表示のみ）│
│              [キャンセル] [計画を作る] │ ← CTA: orange.600
└────────────────────────────────────┘
```

- ボタンは「実行」ではなく **「計画を作る」**: 押してもゲーム生成は始まらず計画承認で止まる、という契約を文言で示す。
- 上限値はここでは変更不可（変更は S3 設定画面＋人間承認。Stage 2）。

## 6. デザイントークンの適用（Orange テンプレ準拠）

- 背景 `#F8F8FB`、カード `#FFFFFF`（radius 8px、shadow subtle）、本文 12–14px `Noto Sans JP`。
- **プライマリ（CTA・選択・フォーカス）**: `orange.600 #FB5B01`。承認バナー・重要 KPI は `orange.900 #AC3E00`。
- 状態色: 成功/承認 `green.600 #259D63`、停止/拒否 `red.600 #FE3939`、進行中 `orange.400 #FF8D44`、非活性 `gray.400 #999999`。
- 色だけで状態を識別しない: バッジは必ずアイコン（●◐✔✖）＋ラベル併記（アクセシビリティ原則）。
- オレンジの多用禁止: タイムラインの通常行はグレー系、オレンジは「ユーザーの行動が必要な箇所」（承認カード・バナー・CTA）に限定する。
